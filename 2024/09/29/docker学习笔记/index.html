解决 docker pull 太慢的方法

在安装好 docker 后，我发现在有 vpn 的情况下，拉取一些小的镜像还是很快速的，但是大的镜像就死活拉取不下来（比如一个完整的 pytorch，最少都有3～4个G）。于是我在 github 上找到了一个库，可以比较好的解决问题（可惜没有完全解决）。

  1. fork 一下这个项目；

  2. 点进自己仓库中的这个项目，找到左上角的 Action，点击进入；

  3. 根据指令集的架构来选择合适的 workflow，一般来说都选择 x86_64；

  4. 点击 Run workflow，并输入要下载的镜像名称，再点击 Run workflow；

  5. 跑完以后点进你的工作流，点击右下角的下载图标，把离线镜像下载下来；

  6. 这个下载下来的文件需要解压两次，假设解压出来为pytorch_pytorch:1.9.0-cuda10.2-cudnn7-devel-amd64.tar，那么将其导入 docker 的指令为

    这样就可以在 docker 上发现这个镜像了！
